## 💡 正确的排查逻辑

### 第一步：确认云函数是否被调用

**方法：查看云函数日志**

1. CloudBase → 云函数 → aiTriage → 日志查询
2. 选择时间范围：最近1小时
3. 查看是否有调用记录

**结果判断：**
- 日志为空 → 云函数未被调用（前端问题）
- 有日志但报错 → 云函数被调用但有问题（后端问题）

### 第二步：确认参数在哪里

**方法：打印完整 event**

```javascript
console.log('完整event:', JSON.stringify(event, null, 2));
```

**从日志中看到 event 的真实结构，才能知道参数在哪里！**

### 第三步：确认 DeepSeek API 是否被调用

**方法：查看 DeepSeek 后台**

- 调用次数是否增加
- 余额是否减少

### 方案B：简化架构（备选）

**放弃 CloudBase，改用纯静态方案**

#### B1. 前端直接调用 DeepSeek API

**优点：**
- 最简单，不需要云函数
- 不需要配置 CloudBase

**缺点：**
- API Key 暴露在前端（安全风险）
- 跨域问题（DeepSeek API 可能不支持 CORS）

**实现：**
```javascript
const response = await fetch('https://api.deepseek.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer sk-xxx',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'deepseek-chat',
    messages: [{role: 'user', content: `症状：${symptom}`}]
  })
});
```

**适用场景：**
- 快速验证功能
- 临时方案

---

#### B2. 使用第三方 Serverless 平台

**选项：**
- Vercel Serverless Functions
- Netlify Functions
- Cloudflare Workers

**优点：**
- 配置简单
- 文档清晰
- 社区支持好

**缺点：**
- 需要重新部署
- 国内访问可能慢

---

#### B3. 自建简单后端

**使用 Express + 部署到腾讯云服务器**

**优点：**
- 完全可控
- 调试方便

**缺点：**
- 需要服务器
- 需要运维

### 方案C：降级方案（保底）

**不用 AI，直接用关键词匹配**

const symptomMap = {
  "头疼": "神经内科",
  "胃疼": "消化内科",
  "咳嗽": "呼吸内科",
  // ... 更多映射

**优点：**

- 成本为零

**缺点：**
- 不够智能

**适用场景：**
